# Shelly

All your newsletters in a nutshell.

Shelly is a personal newsletter aggregator that pulls emails from an AgentMail inbox, extracts content and images, and sends one daily digest email.

AI-generated summaries are mandatory in this project.

## The problem

Most newsletter tools still require one of these tradeoffs:

- cluttering your personal inbox,
- creating another reader account,
- or constantly checking a separate app and having to maintain yet another account.

## The approach

Shelly uses an AgentMail inbox as the intake layer.

1. Subscribe your newsletters to AgentMail.
2. Shelly fetches the day's messages using AgentMail API.
3. It extracts text and images from each newsletter.
4. It sends one combined digest email to your real inbox.

## Why this exists

- Keep your personal inbox clean.
- Subscribe newsletters to AgentMail, not your private mailbox.
- Receive one condensed daily email with key updates.

## Features

- Pulls newsletter emails from AgentMail for a specific date.
- Extracts text, links, and images from HTML newsletter content.
- Builds one digest with all updates.
- AI summaries generated by local Ollama.
- Sends digest via AgentMail messages endpoint.
- Supports dry-run mode to preview output without sending.

## Requirements

- Node.js 20+
- npm
- AgentMail admin API key + admin inbox (for intake and job alerts)
- AgentMail sending API key + sending inbox (for subscriber digest delivery)
- Local Ollama instance with a downloaded model 
- Recommended model : gemma4:31b-cloud (to not overwhelm your machine)

## Setup

1. Install dependencies:

```bash
npm install
```

2. Copy environment template and fill values:

```bash
cp .env.example .env
```

3. Verify the AgentMail values:

- `AGENTMAIL_API_BASE_URL`
- `AGENTMAIL_MESSAGES_PATH`
- `AGENTMAIL_ADMIN_API_KEY`
- `AGENTMAIL_ADMIN_INBOX_ID`
- `AGENTMAIL_SENDING_API_KEY`
- `AGENTMAIL_SENDING_INBOX_ID`
- `AGENTMAIL_*_PARAM` names if your endpoint uses different query names

4. Set recipient value:

- `DIGEST_RECIPIENT_EMAIL`

## Usage

Build digest for today in configured timezone (stores files under output directory, no sending):

```bash
npm run digest
```

Send today's digest email(s) from previously stored artifacts (no generation):

```bash
npm run digest:send
```

Run cleanup independently (deletes artifact files older than retention window):

```bash
npm run digest:cleanup
```

Run digest build for a specific date:

```bash
npm run digest -- --date 2026-04-09
```

Send stored digest for a specific date:

```bash
npm run digest:send -- --date 2026-04-09
```

Dry run (no email sent):

```bash
# Set DIGEST_DRY_RUN=true in .env
npm run digest -- --date 2026-04-09
```

Digest output storage and cleanup:

- `DIGEST_OUTPUT_DIR` controls where daily artifacts are written.
- `DIGEST_OUTPUT_RETENTION_DAYS` controls deletion of old artifact files when `npm run digest:cleanup` is run.
- `DIGEST_SUBSCRIBERS_FILE` points to a JSON subscriber list used by `digest:send`.
- `DIGEST_RECIPIENT_EMAILS` is a fallback comma-separated list if the subscriber file is missing.

Subscriber file format example (`data/subscribers.json`):

```json
[
	"you@example.com",
	"friend@example.com"
]
```

Job status alerts:

- `DIGEST_ALERT_RECIPIENT_EMAIL` sets where status notifications are sent (defaults to `DIGEST_RECIPIENT_EMAIL`).
- `DIGEST_ALERTS_ENABLED=true|false` enables or disables status notification emails.
- Alerts are sent using the AgentMail admin key/inbox channel.
- On any job failure (`digest`, `digest:send`, or `digest:cleanup`), Shelly sends a failure alert with the failed job and error reason.
- On successful end-to-end completion for a date (build completed first, then send completed), Shelly sends a success alert.

## Local AI with Ollama

Shelly requires Ollama summaries. In your local Docker setup:

1. Run Ollama and pull a model, for example `gemma4:31b-cloud`.
2. In `.env` set:

```bash
AI_PROVIDER=ollama
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=gemma4:31b-cloud
OLLAMA_TIMEOUT_MS=90000
OLLAMA_MAX_RETRIES=2
OLLAMA_RETRY_BACKOFF_MS=1500
```

If Ollama is unavailable at runtime, Shelly fails fast so you do not send non-AI summaries.

## Skills and model providers

When this is packaged as a skill, keep the same app and swap only provider config:

- local self-hosted mode: `AI_PROVIDER=ollama`
- managed skill mode: add another provider module and route `AI_PROVIDER` to it

This keeps one codebase while allowing different model backends.

## AgentMail API notes

AgentMail response shapes may vary. The parser handles common field names for:

- message id
- sender/from
- subject
- text body
- html body
- attachment image URLs

If your endpoint returns a different shape, update mapping logic in `src/agentmailClient.ts`.

## Changesets and versioning

Shelly uses Changesets to keep upgrade history in files.

Create a changeset for any user-visible or behavior change:

```bash
npm run changeset
```

Apply pending changesets to versions/changelog:

```bash
npm run version-packages
```

Publish step (if you later publish packages):

```bash
npm run release
```
